# Les outils du développeur

## Installations initiales

- [Windows](/installations/windows)
- [MacOS](/installations/macos)
- [Ubuntu](/installations/ubuntu)

Voici la liste des outils recommandés :

- [**WSL**](https://learn.microsoft.com/fr-fr/windows/wsl/install) *(Windows uniquement)* — Windows Subsystem for Linux pour exécuter un environnement Linux sur Windows
- [**Homebrew**](https://brew.sh/) *(macOS uniquement)* — Gestionnaire de paquets pour macOS (équivalent de `apt`)
- [**Git**](https://git-scm.com/) * — Système de contrôle de version décentralisé, utilisé avec GitHub pour tous les projets
- [**zsh**](https://doc.ubuntu-fr.org/zsh) * — Shell Unix moderne et personnalisable (shell par défaut sur macOS)
- [**oh-my-zsh**](https://ohmyz.sh/) * — Framework de configuration pour zsh avec thèmes, plugins et raccourcis
- [**proto**](https://moonrepo.dev/docs/proto) * — Gestionnaire de versions multi-langages unifié (Node.js, Python, Go...) avec `.prototools`
- [**Node.js**](https://nodejs.org/fr) *(par proto de préférence)* — Runtime JavaScript côté serveur (**LTS 24.x recommandé**)
- [**GitHub CLI**](https://cli.github.com/) * — Outil CLI pour interagir avec GitHub (PR, issues, repos...)
- [**pnpm**](https://pnpm.io/) * — Gestionnaire de dépendances Node.js performant (**v10.x recommandé**)
- [**Docker**](https://www.docker.com/) — Plateforme de conteneurisation pour le développement et le déploiement
- [**uv**](https://docs.astral.sh/uv/) *(projets Python)* — Gestionnaire de projets Python ultra-rapide avec `pyproject.toml` et `uv.lock`


\* Pour Windows, doit être installé dans WSL

Plus de détails pour les installations dans les pages dédiées à chaque OS.
